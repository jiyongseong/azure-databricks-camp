# Lakehouse

Databricks 데이터 인텔리전스 플랫폼을 이야기할 때, 그 기반에는 통합 Lakehouse 플랫폼이 있습니다.

## Data Lake와 Data Warehouse
<img src='https://www.databricks.com/wp-content/uploads/2020/01/data-lakehouse-new.png'>

레이크하우스(Lakehouse)는 Databricks가 제안한 비교적 새로운 데이터 플랫폼 카테고리로, 대략 4년 전쯤 관련 백서와 학술 논문과 함께 대중화되기 시작했다는 점에서 출발합니다. 

이 개념은 과거 데이터 레이크와 데이터 웨어하우스가 각자 잘하는 영역은 뚜렷하지만, 서로의 약점을 보완하지 못해 생기던 실무적 문제를 해결하려는 시도입니다. 

시간을 2012년 전후로 돌려보면, 당시엔 Hadoop과 MapReduce가 사실상의 표준이었고 Spark가 막 존재감을 드러내던 시기였습니다. 

그 무렵 등장한 데이터 레이크는 클라우드 오브젝트 스토리지를 중심에 두고 있었습니다. 오브젝트 스토리지는 매우 내구성이 높고 비용 효율적이며, 수백만 개의 파일을 다루는 데 최적화되어 있었기에, 기업들은 "모든 데이터를 한 곳에 모아두면 모두가 접근할 수 있다"는 믿음으로 방대한 원천 데이터를 그곳에 쌓았습니다.

하지만 시간이 흐르면서 데이터 레이크는 종종 **데이터 스왐프(data swamp)**로 변했습니다. 무엇이 어디에 있는지 파악하기 어렵고, 통일된 메타데이터와 거버넌스가 약하며, 보안 모델도 파일 단위에 머무르는 경우가 많아 파일 내부 레코드 수준의 세분화된 접근 제어를 구현하기 어려웠습니다. 

예컨대 오브젝트 스토리지에 고객 데이터가 담긴 Parquet 파일이 있을 때, JYSEONG라는 사용자는 "경기 지역의 고객 데이터만 보게 한다"와 같은 정교한 정책을 일관되게 적용하기가 쉽지 않았습니다. 

그럼에도 레이크의 장점은 분명했다. 머신러닝은 데이터가 많을수록 성능이 좋아지는 경향이 있기 때문에, 비정형 데이터를 대량 저장, 활용할 수 있는 레이크는 고급 ML 활용에 유리했습니다. 다만 레이크는 SQL 기반의 애드혹 질의 성능과 일관된 보안·거버넌스 측면에서 한계를 드러냈습니다.

반대로, 데이터 웨어하우스는 40년 넘게 성숙해 온 기술로, BI와 리포팅에서는 여전히 최고 수준의 안정성과 성능, 그리고 강력한 보안, 거버넌스를 제공합니다. 그러나 비정형 데이터 처리나 머신러닝·실험 반복 같은 현대적 워크로드에는 제약이 많습니다. 결국 조직은 같은 데이터를 두 체계에 나눠 담고 복제/이동/동기화/중복 관리라는 운영 비용을 떠안게 되었습니다.

## Lakehouse의 등장
레이크하우스는 이 간극을 메우기 위해 등장했습니다. 핵심 아이디어는 간단합니다. 데이터 레이크의 개방성/확장성/저비용 저장과 데이터 웨어하우스의 관리/일관성/SQL 분석 성능을 한 시스템 안에 결합해, 각자의 약점을 상쇄하는 것입니다. 

이렇게 되면 하나의 공통 저장 계층(주로 오브젝트 스토리지) 위에 신뢰 가능한 테이블 추상화와 트랜잭션/스키마 관리/일관된 권한체계를 부여해, BI/애드혹 SQL/데이터 과학/머신러닝/스트리밍까지 단일한 진실의 원천(Single Source of Truth) 에서 처리할 수 있습니다.

결과적으로 조직은 데이터 복제와 사본 관리의 복잡성을 줄이고, 보안/거버넌스를 중앙집중적으로 적용하며, 분석과 ML 팀이 동일한 데이터 맥락을 공유하도록 만들 수 있습니다. 

요컨대 레이크하우스는 레이크와 웨어하우스의 장점을 합성하고, **"데이터는 한 번 저장하고 여러 방식으로 활용한다"** 는 현대 데이터 전략을 현실화하기 위한 구조적 해결 방법입니다.

## Lakehouse == Single Source of Truth
Databricks는 진정한 Lakehouse 플랫폼입니다.

이 플랫폼은 스토리지, 처리, 거버넌스, 데이터 공유를 모두 하나의 통합 아키텍처에서 제공합니다. 사용자는  Databricks에 로그인하기만 하면 됩니다.
여러 개의 웹 서비스를 따로 관리할 필요가 없습니다.

또한 Lakehouse는 다양한 방식으로 사용될 수 있습니다.
BI/데이터 사이언스/머신러닝/스트리밍/생성형 AI까지 다양한 방식으로 사용이 가능합니다.

**Lakehouse의 진정한 강점은 모든 사용자가 동일한 데이터를 본다는 점** 입니다.
BI 담당자, 리포트 작성자, 데이터 사이언티스트, 머신러닝 엔지니어 모두 동일한 데이터에 접근합니다.
즉, **단일 진실 공급원(Single Source of Truth)이 보장** 됩니다.

리포트를 위해 데이터 레이크에서 데이터 웨어하우스로 데이터를 옮길 필요가 없습니다. 모든 데이터는 클라우드 오브젝트 스토리지에 저장됩니다. 구조화 데이터, 비정형 데이터, 반정형 데이터 모두 포함됩니다. 심지어 머신러닝 모델도 계정 내 클라우드 오브젝트 스토리지에 저장됩니다.
이것이 Lakehouse의 핵심입니다.

Databricks가 고객에게 제공하는 비즈니스 가치는 바로 Lakehouse를 통해 실현됩니다. 그리고 그 핵심에는 Delta Lake가 있습니다. 모든 고객, 모든 직원은 하나의 플랫폼에 로그인하여 모든 작업을 수행할 수 있습니다.
우리는 모든 활용 사례와 모든 사용자 페르소나를 지원합니다.
